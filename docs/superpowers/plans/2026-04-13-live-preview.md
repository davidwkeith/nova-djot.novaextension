# Djot Live Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a live preview that renders Djot to HTML in a split pane, updated in real-time via WebSocket, with editor-to-preview scroll sync.

**Architecture:** Extend the existing `djot-lsp` Go binary with an HTTP server (random localhost port) and WebSocket endpoint. The HTTP server serves a self-contained HTML preview page. On every `didChange`, the server renders Djot to HTML using godjot's `ConvertDjotToHtml` and pushes the HTML over WebSocket. Scroll sync sends cursor line numbers from Nova JS to the preview page, which scrolls to the matching `data-line` element.

**Tech Stack:** Go (net/http, gorilla/websocket, godjot html_writer), Nova LanguageClient API, JavaScript WebSocket

**Spec:** `docs/superpowers/specs/2026-04-13-live-preview-design.md`

**Security note:** The preview page renders trusted content from the local editor via a local-only WebSocket connection. The HTML is generated by godjot from the user's own Djot source. The preview page is served on localhost and is not accessible from external networks. The content is set via DOM manipulation of a trusted pipeline (user source -> godjot renderer -> local WebSocket -> preview page).

```go
func TestRenderHTML_BasicParagraph(t *testing.T) {

	src := "# Hello\n"
	doc := NewDocument("file:///test.dj", src)
	html := renderHTML(doc)

	if !strings.Contains(html, "<h1") {
		t.Errorf("expected <h1> tag, got %q", html)
	}
	if !strings.Contains(html, "Hello") {
		t.Errorf("expected heading text, got %q", html)
	}

func TestRenderHTML_CodeBlock(t *testing.T) {
	src := "```python\nprint(\"hi\")\n```\n"
	doc := NewDocument("file:///test.dj", src)
	html := renderHTML(doc)

	if !strings.Contains(html, "<pre") || !strings.Contains(html, "<code") {
		t.Errorf("expected pre/code tags, got %q", html)
	}
}

func TestRenderHTML_EmptyDocument(t *testing.T) {
	doc := NewDocument("file:///test.dj", "")
	html := renderHTML(doc)

	if html != "" {
		t.Errorf("expected empty string for empty doc, got %q", html)
	}
}

	doc := NewDocument("file:///test.dj", src)

		t.Errorf("expected at least 3 data-line attributes, got %d in %q", count, html)
```

- [ ] **Step 2: Add HTTP server and WebSocket hub to preview.go**

Append to `preview.go`:

- `PreviewServer` struct — holds `clients map[*websocket.Conn]bool`, `port int`, `server *http.Server`, `upgrader websocket.Upgrader`, `mu sync.RWMutex`
- `var preview *PreviewServer` — package-level singleton
- `StartPreviewServer() (int, error)` — listens on `localhost:0`, creates mux with `/preview` and `/ws` handlers, starts `Serve` in a goroutine, returns port
- `StopPreviewServer()` — calls `server.Shutdown`
- `BroadcastContent(doc *Document)` — renders HTML with line numbers, sends JSON `{"type":"content","html":"..."}` to all WebSocket clients
- `BroadcastScroll(line int)` — sends JSON `{"type":"scroll","line":N}` to all clients
- `handleWebSocket` — upgrades connection, adds to clients, read loop (discards messages), removes on close
- `handlePreviewPage` — serves the self-contained HTML page with embedded CSS and JS
- `jsonEscape(s string) string` — escapes a string for embedding in JSON
- `previewPageHTML(port int) string` — returns the full HTML page as a string

The preview HTML page includes:
- Clean CSS styling (system font, code blocks, tables, blockquotes, dark mode support via `prefers-color-scheme`)
- WebSocket client JS that connects to `ws://localhost:{port}/ws`
- Auto-reconnect on disconnect (1 second retry)
- On `"content"` message: stores scroll position, updates `#content` element using DOM APIs, restores scroll position
- On `"scroll"` message: finds closest `[data-line]` element where `data-line <= target`, scrolls it into view smoothly
- "Reconnecting..." banner shown when disconnected

The HTML content update in the preview page JS should use safe DOM manipulation. Since the content comes from a trusted local pipeline (user's own Djot source rendered by godjot, delivered over localhost-only WebSocket), use `document.getElementById("content").innerHTML = msg.html` for the content area. This is safe because:
1. The WebSocket only accepts connections from localhost
2. The HTML is generated by godjot from the user's own document
3. There is no external input pathway

- [ ] **Step 1: Update main.js**

Replace `Scripts/main.js` with updated version that:

1. Listens for `djot/previewReady` notification and stores the port
2. Registers command `io.dwk.djot.preview` that opens `http://localhost:{port}/preview` via `nova.workspace.openURL()`
3. On preview open, hooks `nova.workspace.onDidChangeActiveTextEditor` and `editor.onDidChangeSelection` to track cursor movement
4. Sends `djot/scrollTo` custom LSP notification with `{ line: N }` when cursor moves in a Djot file
5. Debounces scroll notifications (only send after 100ms of no cursor movement)

Since Nova JS doesn't have a WebSocket client, scroll sync goes: Nova JS -> LSP notification `djot/scrollTo` -> Go server -> WebSocket broadcast to preview page.

```go
func BroadcastScroll(line int) {
	if preview == nil {
		return
	}
	msg := fmt.Sprintf(`{"type":"scroll","line":%d}`, line)
	preview.mu.RLock()
	defer preview.mu.RUnlock()
	for client := range preview.clients {
		client.WriteMessage(websocket.TextMessage, []byte(msg))
	}
}
```
